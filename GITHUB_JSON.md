# Інструкція: як передати JSON-файли на GitHub

Ця інструкція для користувача, який оновлює новини, фотографії або розклад через адмінку сайту.

## 1. Підготуйте JSON в адмінці

1. Відкрийте адмінку сайту:

```text
admin/index.html
```

або локально:

```text
http://127.0.0.1:8000/admin/index.html
```

2. Введіть пароль адміністратора.
3. Дочекайтеся, поки в блоці `Попередній перегляд` з'являться поточні записи з файлів `data`.
4. Додайте потрібні матеріали:
   - новини;
   - фотографії;
   - розклад.
5. У блоці `Збереження` натисніть потрібну кнопку:
   - `news.json` - для новин;
   - `photos.json` - для фотографій;
   - `schedule.json` - для розкладу.

Після натискання браузер завантажить JSON-файл на комп'ютер. Файл міститиме старі записи та нові додані матеріали.

Важливо: відкривайте адмінку через локальний сервер або GitHub Pages, а не простим подвійним кліком по файлу. Інакше браузер може заблокувати читання поточних JSON-файлів.

## 2. Замініть файл у папці `data`

У проєкті є папка:

```text
data
```

У ній лежать три файли:

```text
news.json
photos.json
schedule.json
```

Замініть старий файл новим завантаженим файлом:

- якщо завантажили `news.json`, замініть `data/news.json`;
- якщо завантажили `photos.json`, замініть `data/photos.json`;
- якщо завантажили `schedule.json`, замініть `data/schedule.json`.

Назва файлу має залишитися точно такою самою.

## 3. Перевірте сайт локально

Якщо сайт відкритий локально, оновіть сторінку в браузері.

Перевірте відповідний розділ:

- `Новини`;
- `Фотографії`;
- `Розклад`.

Якщо дані відображаються правильно, можна передавати зміни на GitHub.

## 4. Відправте зміни на GitHub через VS Code

1. Відкрийте проєкт у VS Code.
2. Перейдіть у розділ `Source Control`.
3. Переконайтеся, що у змінах є потрібний файл з папки `data`.
4. У полі повідомлення напишіть короткий опис, наприклад:

```text
update news
```

або:

```text
update photos
```

5. Натисніть `Commit`.
6. Натисніть `Sync Changes` або `Push`.

Після цього зміни потраплять на GitHub.

## 5. Відправте зміни через термінал

Якщо зручно працювати через термінал, виконайте одну з команд.

Для новин:

```bash
git add data/news.json
git commit -m "update news"
git push
```

Для фотографій:

```bash
git add data/photos.json
git commit -m "update photos"
git push
```

Для розкладу:

```bash
git add data/schedule.json
git commit -m "update schedule"
git push
```

Якщо оновлено кілька файлів одразу:

```bash
git add data/news.json data/photos.json data/schedule.json
git commit -m "update site data"
git push
```

## 6. Перевірте GitHub Pages

Після `push` GitHub Pages може оновлювати сайт не миттєво.

Зазвичай потрібно зачекати від кількох секунд до кількох хвилин, потім оновити сторінку сайту в браузері.

## Важливо

- Не змінюйте назви файлів `news.json`, `photos.json`, `schedule.json`.
- Не переміщуйте ці файли з папки `data`.
- Якщо після оновлення сайт не показує нові дані, перевірте, чи був виконаний `git push`.
- Якщо фотографії не відображаються, перевірте, чи відкриваються Cloudinary-посилання в окремій вкладці браузера.
